# Foundry

An autonomous pipeline that generates, builds, and deploys micro-SaaS products using Claude Cowork (ideas + specs) and Claude Code (implementation).

## How It Works

```
Claude Cowork          Claude Code            GitHub + Vercel
┌──────────┐          ┌──────────┐          ┌──────────────┐
│ Generate  │    →     │  Build   │    →     │ Push + Deploy│
│ idea +    │          │  from    │          │              │
│ spec      │          │  spec    │          │ Update       │
│           │    →     │  Polish  │    →     │ portfolio    │
│           │          │  UI pass │          │              │
└──────────┘          └──────────┘          └──────────────┘
```

## Setup

### Prerequisites

```bash
# Claude Code CLI
# (install from https://docs.claude.com)

# GitHub CLI
brew install gh
gh auth login

# Vercel CLI
npm i -g vercel
vercel login

# jq (for JSON processing)
brew install jq
```

### Make scripts executable

```bash
cd ~/Code/exploratory/foundry
chmod +x build-project.sh run-factory.sh
```

### Deploy the portfolio page

```bash
cd ~/Code/exploratory/foundry
git init
gh repo create foundry --public --source=. --push
vercel --prod --yes
```

Save the Vercel URL — that's your portfolio page.

## Usage

### Build a single project (spec already exists)

```bash
./build-project.sh demoseed
./build-project.sh isitup
```

This runs the full pipeline:
1. Feeds the spec to Claude Code (initial build)
2. Runs a second Claude Code pass (UI polish)
3. Creates a GitHub repo and pushes
4. Deploys to Vercel
5. Updates `projects.json` with live URL
6. Pushes portfolio update

### Generate new ideas + build them

```bash
# Generate 3 new ideas, create specs, then build all
./run-factory.sh --generate 3

# Just build whatever's queued
./run-factory.sh --build-only
```

### Run the full 24/7 loop

```bash
# In a tmux/screen session:
while true; do
  ./run-factory.sh --generate 2
  sleep 300  # wait 5 min between runs
done
```

## projects.json Schema

```json
{
  "projects": [
    {
      "id": "project-slug",
      "name": "Display Name",
      "tagline": "One-liner under 60 chars",
      "description": "2-3 sentence description",
      "status": "queued | building | built | live | failed",
      "spec_dir": "folder-name",
      "repo": "https://github.com/user/repo",
      "live_url": "https://project.vercel.app",
      "pricing": "$XX/mo",
      "category": "Category Name",
      "color": "#hex",
      "built_at": "YYYY-MM-DD"
    }
  ],
  "queue": []
}
```

## Status Meanings

| Status | Meaning |
|--------|---------|
| `queued` | Spec exists, waiting to be built |
| `building` | Claude Code is currently building it |
| `built` | Code is done, not yet deployed |
| `live` | Deployed to Vercel with a live URL |
| `failed` | Build failed — check logs |
